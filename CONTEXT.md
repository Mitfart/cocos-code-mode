# Context

## What this is

`cocos-code-mode` is a Cocos Creator extension. It runs an Express/UTCP server inside the editor and registers a dynamic `CocosEditor` provider so AI agents can inspect and mutate Cocos projects through editor APIs instead of raw file edits.

## Domain terms

- **Code Mode**: agent pattern where the AI writes compact JavaScript against tool bindings instead of calling many individual tools.
- **UTCP**: protocol used to publish tool manuals and call templates.
- **CocosEditor provider**: the live UTCP provider generated by this extension. Port is dynamic.
- **Editor-owned data**: Cocos-managed scene/prefab/material/asset/meta files. Must be changed through editor APIs.
- **Instance reference**: temporary handle like `{ id, type }` returned by tree/component/asset queries. Do not persist across sessions.
- **Discover, then act**: query hierarchy/properties/definitions before mutations. Never guess property paths.

## Architecture

- `source/main.ts`: extension lifecycle, panel commands, server start/restart, UTCP config update.
- `source/scene.ts`: scene-process helpers for prefab, screenshot, logging.
- `source/utcp/*`: UTCP server, schemas, decorators, config manager.
- `dist/*`: build output. Do not edit manually.

## Current decisions

- Dynamic port is normal; config update is part of startup.
- Build output lives in `dist/`; source of truth is `source/`.
- Use ADRs only for surprising or hard-to-reverse design choices.
